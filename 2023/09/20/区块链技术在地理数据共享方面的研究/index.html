区块链技术在地理数据共享方面的研究

发表于2023-09-20|更新于2024-02-22
|阅读量:

文献综述

[1]曹美荣,李雅娜.基于区块链的电子数据存证应用研究[J].信息技术与信息化,2023(12):177-180.

将电子数据存证与区块链进行融合，构建基于HyperLedger Fabric架构的区块链电子数据存证方案。
采用IPFS用来解决区块链存储大量非结构化数据的问题，如图片、音频、视频、网页、文书文档等。具体来说，将原始文件存储在IPFS上，然后对文件进行哈希计算后的Hash值上传到区块链进行存储。这样可以减少区块链的存储空间，降低电子数据文件的存储成本，提高区块链的运行效率和存储性能，同时解决了数据的完整性、安全性、隐私性问题。

[1]苏猛猛,胡满,张勇等.基于区块链的电力系统稳控数据存证与共享技术研究[J].电力信息与通信技术,2024,22(01):93-99.DOI:10.16543/j.2095-641x.electric.power.ict.2024.01.12.

从区块链技术特点出发，深入分析了稳控系统与区块链深度应用融合的可行性。然后，他们开展了基于区块链的稳控系统数据存证与共享应用架构的研究，设计了区块数据结构和一种股份授权证明共识机制。通过实际应用案例，对稳控数据在上链存证共享、数据同步共识等应用性能进行了分析。

[1]张新,谭学泽,王小艺等.基于区块链和联邦学习的食品全程全息风险信息可信共享模式[J/OL].食品科学:1-26[2024-02-21].http://kns.cnki.net/kcms/detail/11.2206.TS.20240130.0921.008.html.

区块链技术

区块链具有去中心化分布式账本，点对点传输，加密算法，智能合约，共识机制等特性。
区块链的本质是什么

密码学原理

哈希SHA-256

collision resistance
哈希碰撞x≠y但H(x)=H(y)
hiding
x能得到H(x)但不能通过H(x)反推x
puzzle friendly

签名

开户创建一个公钥私钥的对。
用私钥签名，其他人用公钥验证签名

数据结构

哈希指针存地址和哈希值，代替普通指针
区块链改任何一个后面hash值都会改变
merkle tree

block header：
version机制版本
hash of previous block header前一区块头哈希
merkle root hash树根哈希值
time时间
target目标阈值
nonce随机数
block body：
transaction list交易列表

共识协议

双花攻击double spending attack
女巫攻击sybil attack
分叉攻击forking attack

接受区块应该在拓展最长合法链longest valid chain
bitcoin中按算力来争取记账权，可以决定交易区块，可以奖励币，争夺记账权的过程称为挖矿

实现

UTXO：unspent transaction output检测没花出去的币，防止双花

比特币transaction-based ledger
以太坊account-based ledger

激励机制：出块奖励和交易费

memoryless
progress free保证挖矿公平性

geometric series
一共2100万比特币

网络

application layer：BitCoin Block chain
network layer：P2P Overlay Network
原则：简单，强大，但不是高效

挖矿

全节点和轻节点
无记忆性，难度一样
通用CPU挖矿-通用并行GPU挖矿-专用ASIC芯片挖矿

工作量证明：share（almost valid block）矿主在矿池中看每个矿工提交的share数量作为工作量衡量直到挖到合法区块

区块链编写

使用JavaScript
构建区块（时间，数据，自己的哈希值，前一个区块的哈希值）Block类
区块成链（创世区块）BlockChain类
实现工作证明（区块挖矿）nonce值
